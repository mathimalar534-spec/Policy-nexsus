* **Technical Rationale (NIST SP 800-63B Guidance):**
  * **Reduction of User Fatigue:** NIST guidelines specifically advise *against* forcing periodic password changes (e.g., every 90 days) unless there is explicit evidence of compromise. Frequent resets prompt users to select weaker, easily guessable passwords (e.g., incrementing a trailing number) or write them down, which actively degrades the enterprise security perimeter.
  * **Focus on Modern Authentication:** By migrating legacy rotation controls to a passwordless Multi-Factor Authentication (MFA) environment and utilizing phishing-resistant verifiers, the enterprise decreases exposure to credential stuffing and phishing attacks while streamlining the user experience.
  * **Implementation:** Disable the legacy 90-day automatic password expiry prompts on Active Directory, Entra ID, and Okta platforms globally.
