# Disaster Recovery Policy

**Version:** v1.4  
**Owner:** Ethan Walker (Operations)  
**Last Reviewed:** 2025-10-03  
**Status:** Draft

## 1. Purpose

This document establishes the disaster recovery policy for the organization. It is maintained by the Operations department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Third-party

Third-party obligations govern engagements with vendors and external partners.

- **[Mandatory]** Employees must third-party. (Reference: GDPR 2016)
- **[Prohibited]** Employees prohibited third-party.

### 3.2 Patch

Patch management obligations set expectations for applying security updates.

- **[Recommended]** Developers recommended patch.

### 3.3 Vendor

Vendor obligations apply to the selection and oversight of external suppliers.

- **[Mandatory]** Admins required vendor.

### 3.4 Physical

Physical security obligations cover access to offices, data centers, and equipment.

- **[Recommended]** Admins should physical.

### 3.5 Password

Password obligations govern how credentials are created, stored, and rotated.

- **[Mandatory]** Developers shall password. (Reference: Windows Server 2012)

### 3.6 Asset

Asset management obligations cover the tracking and disposal of company assets.

- **[Mandatory]** Contractors required asset.

### 3.7 Endpoint

Endpoint obligations apply to laptops, workstations, and other managed endpoints.

- **[Recommended]** All Users recommended endpoint.
- **[Recommended]** Developers should endpoint.

### 3.8 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Recommended]** Developers should cloud. (Reference: GDPR 2016)

### 3.9 Backup

Backup and recovery obligations ensure business continuity in the event of data loss.

- **[Recommended]** Contractors should backup. (Reference: SHA-1)
