# Vendor Management Policy

**Version:** v2.3  
**Owner:** Ethan Walker (Operations)  
**Last Reviewed:** 2024-12-07  
**Status:** Active

## 1. Purpose

This document establishes the vendor management policy for the organization. It is maintained by the Operations department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Mandatory]** All Users must cloud.
- **[Mandatory]** Contractors shall cloud. (Reference: Windows Server 2012)
- **[Recommended]** All Users recommended cloud.

### 3.2 Monitoring

Monitoring obligations define what systems must be observed and alerted on.

- **[Recommended]** Employees should monitoring.

### 3.3 HR

HR obligations apply to onboarding, offboarding, and personnel security practices.

- **[Recommended]** All Users recommended HR. (Reference: FTP)

### 3.4 Change

Change management obligations govern how modifications to production systems are approved.

- **[Recommended]** Admins should change. (Reference: FTP)

### 3.5 Password

Password obligations govern how credentials are created, stored, and rotated.

- **[Recommended]** Admins should password.
- **[Mandatory]** Developers must password.

### 3.6 Physical

Physical security obligations cover access to offices, data centers, and equipment.

- **[Mandatory]** Developers must physical.
- **[Prohibited]** Employees prohibited physical.

### 3.7 Encryption

Encryption obligations specify how data must be protected at rest and in transit.

- **[Mandatory]** Developers shall encryption.

### 3.8 Patch

Patch management obligations set expectations for applying security updates.

- **[Mandatory]** Employees required patch. (Reference: SOX 2002)

## 4. References

This policy's technical baseline is defined with reference to the pre-2018 draft of the EU Data Protection Directive (superseded by GDPR), as adopted at the time of the last major revision.
