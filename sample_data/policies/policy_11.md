# Remote Work Policy

**Version:** v3.2  
**Owner:** Daniel Kim (Product)  
**Last Reviewed:** 2025-06-02  
**Status:** Draft

## 1. Purpose

This document establishes the remote work policy for the organization. It is maintained by the Product department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Vendor

Vendor obligations apply to the selection and oversight of external suppliers.

- **[Mandatory]** Admins required vendor.
- **[Prohibited]** Employees prohibited vendor.

### 3.2 Encryption

Encryption obligations specify how data must be protected at rest and in transit.

- **[Mandatory]** Admins required encryption. (Reference: DES)

### 3.3 Asset

Asset management obligations cover the tracking and disposal of company assets.

- **[Mandatory]** Admins must asset.
- **[Prohibited]** Admins prohibited asset. (Reference: DES)

### 3.4 Access

Access control obligations define who may reach which systems and data.

- **[Mandatory]** Admins required access.

### 3.5 Network

Network obligations govern segmentation, firewalling, and perimeter defenses.

- **[Prohibited]** All Users prohibited network. (Reference: GDPR 2016)

### 3.6 Third-party

Third-party obligations govern engagements with vendors and external partners.

- **[Mandatory]** All Users must third-party.

### 3.7 Data retention

Data retention obligations set minimum and maximum retention periods for records.

- **[Mandatory]** All Users must data retention.

### 3.8 Mobile

Mobile device obligations apply to any device used to access company resources.

- **[Prohibited]** Developers prohibited mobile.

### 3.9 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Mandatory]** Contractors required cloud.

### 3.10 Monitoring

Monitoring obligations define what systems must be observed and alerted on.

- **[Mandatory]** All Users must monitoring.
