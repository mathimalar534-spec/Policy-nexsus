# Encryption Policy

**Version:** v3.3  
**Owner:** Daniel Kim (Infrastructure)  
**Last Reviewed:** 2023-04-07  
**Status:** Active

## 1. Purpose

This document establishes the encryption policy for the organization. It is maintained by the Infrastructure department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Encryption

Encryption obligations specify how data must be protected at rest and in transit.

- **[Mandatory]** Contractors shall encryption.

### 3.2 Network

Network obligations govern segmentation, firewalling, and perimeter defenses.

- **[Mandatory]** Employees required network.
- **[Recommended]** Admins recommended network.

### 3.3 Provisioning

Provisioning obligations define how user accounts are created and deprovisioned.

- **[Prohibited]** Developers prohibited provisioning.

### 3.4 Logging

Logging obligations define what activity must be recorded and for how long.

- **[Mandatory]** Employees must logging.

### 3.5 Privacy

Privacy obligations govern the collection, use, and disclosure of personal data.

- **[Mandatory]** Contractors must privacy.

### 3.6 Monitoring

Monitoring obligations define what systems must be observed and alerted on.

- **[Mandatory]** Employees must monitoring.

### 3.7 Mobile

Mobile device obligations apply to any device used to access company resources.

- **[Mandatory]** Employees must mobile.
- **[Mandatory]** Admins must mobile.

### 3.8 Third-party

Third-party obligations govern engagements with vendors and external partners.

- **[Prohibited]** All Users prohibited third-party.

### 3.9 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Recommended]** Developers recommended cloud.

### 3.10 Endpoint

Endpoint obligations apply to laptops, workstations, and other managed endpoints.

- **[Mandatory]** Developers required endpoint.
