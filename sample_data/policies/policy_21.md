# Logging Policy

**Version:** v1.0  
**Owner:** Priya Nair (Infrastructure)  
**Last Reviewed:** 2025-05-02  
**Status:** Retired

## 1. Purpose

This document establishes the logging policy for the organization. It is maintained by the Infrastructure department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Endpoint

Endpoint obligations apply to laptops, workstations, and other managed endpoints.

- **[Recommended]** Contractors should endpoint.

### 3.2 Third-party

Third-party obligations govern engagements with vendors and external partners.

- **[Prohibited]** All Users prohibited third-party.

### 3.3 Mobile

Mobile device obligations apply to any device used to access company resources.

- **[Recommended]** Employees should mobile.

### 3.4 Backup

Backup and recovery obligations ensure business continuity in the event of data loss.

- **[Mandatory]** Developers required backup.

### 3.5 Password

Password obligations govern how credentials are created, stored, and rotated.

- **[Mandatory]** All Users required password. (Reference: SHA-1)

### 3.6 Patch

Patch management obligations set expectations for applying security updates.

- **[Mandatory]** All Users must patch.

### 3.7 Access

Access control obligations define who may reach which systems and data.

- **[Recommended]** Contractors recommended access.
- **[Recommended]** Developers should access.

### 3.8 Change

Change management obligations govern how modifications to production systems are approved.

- **[Recommended]** Contractors recommended change.

### 3.9 Encryption

Encryption obligations specify how data must be protected at rest and in transit.

- **[Recommended]** Contractors recommended encryption.

### 3.10 Monitoring

Monitoring obligations define what systems must be observed and alerted on.

- **[Mandatory]** Admins shall monitoring.

## 4. References

This policy's technical baseline is defined with reference to the 1996 HIPAA Security Rule guidance (pre-Omnibus Rule), as adopted at the time of the last major revision.
