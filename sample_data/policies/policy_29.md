# Endpoint Security Policy

**Version:** v1.3  
**Owner:** Alex Morgan (Compliance)  
**Last Reviewed:** 2025-07-26  
**Status:** Active

## 1. Purpose

This document establishes the endpoint security policy for the organization. It is maintained by the Compliance department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Provisioning

Provisioning obligations define how user accounts are created and deprovisioned.

- **[Mandatory]** Developers required provisioning.

### 3.2 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Prohibited]** Admins prohibited cloud.

### 3.3 Network

Network obligations govern segmentation, firewalling, and perimeter defenses.

- **[Mandatory]** Employees required network.

### 3.4 Vendor

Vendor obligations apply to the selection and oversight of external suppliers.

- **[Mandatory]** Contractors shall vendor.

### 3.5 Access

Access control obligations define who may reach which systems and data.

- **[Mandatory]** All Users required access.
- **[Prohibited]** Employees prohibited access.

### 3.6 Physical

Physical security obligations cover access to offices, data centers, and equipment.

- **[Mandatory]** Employees must physical.

### 3.7 Mobile

Mobile device obligations apply to any device used to access company resources.

- **[Prohibited]** Contractors prohibited mobile.

### 3.8 Data retention

Data retention obligations set minimum and maximum retention periods for records.

- **[Recommended]** Admins recommended data retention.

### 3.9 Monitoring

Monitoring obligations define what systems must be observed and alerted on.

- **[Mandatory]** Contractors shall monitoring.

### 3.10 HR

HR obligations apply to onboarding, offboarding, and personnel security practices.

- **[Prohibited]** Contractors prohibited HR.
