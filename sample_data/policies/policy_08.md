# Access Control Policy

**Version:** v3.2  
**Owner:** Alex Morgan (Support)  
**Last Reviewed:** 2023-01-23  
**Status:** Active

## 1. Purpose

This document establishes the access control policy for the organization. It is maintained by the Support department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, employees.

## 3. Obligations

### 3.1 Mobile

Mobile device obligations apply to any device used to access company resources.

- **[Prohibited]** All Users prohibited mobile.
- **[Mandatory]** Admins shall mobile.
- **[Mandatory]** All Users shall mobile.

### 3.2 Endpoint

Endpoint obligations apply to laptops, workstations, and other managed endpoints.

- **[Prohibited]** Admins prohibited endpoint.

### 3.3 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Mandatory]** All Users required cloud.

### 3.4 Vendor

Vendor obligations apply to the selection and oversight of external suppliers.

- **[Mandatory]** All Users required vendor.

### 3.5 Data retention

Data retention obligations set minimum and maximum retention periods for records.

- **[Mandatory]** Employees must data retention.

### 3.6 Privacy

Privacy obligations govern the collection, use, and disclosure of personal data.

- **[Mandatory]** Employees required privacy.
- **[Recommended]** All Users should privacy.

### 3.7 Network

Network obligations govern segmentation, firewalling, and perimeter defenses.

- **[Mandatory]** Contractors must network. (Reference: SHA-1)

### 3.8 Physical

Physical security obligations cover access to offices, data centers, and equipment.

- **[Mandatory]** Admins required physical.
- **[Mandatory]** Admins shall physical.
