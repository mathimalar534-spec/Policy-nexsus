# Legal Compliance Policy

**Version:** v3.5  
**Owner:** Alex Morgan (Operations)  
**Last Reviewed:** 2025-07-22  
**Status:** Active

## 1. Purpose

This document establishes the legal compliance policy for the organization. It is maintained by the Operations department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Logging

Logging obligations define what activity must be recorded and for how long.

- **[Mandatory]** Developers must logging.

### 3.2 Endpoint

Endpoint obligations apply to laptops, workstations, and other managed endpoints.

- **[Mandatory]** All Users shall endpoint.

### 3.3 Mobile

Mobile device obligations apply to any device used to access company resources.

- **[Mandatory]** Contractors shall mobile. (Reference: FTP)

### 3.4 Third-party

Third-party obligations govern engagements with vendors and external partners.

- **[Mandatory]** Employees must third-party.

### 3.5 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Mandatory]** Employees required cloud.

### 3.6 Encryption

Encryption obligations specify how data must be protected at rest and in transit.

- **[Recommended]** Admins should encryption.
- **[Mandatory]** All Users must encryption.

### 3.7 Backup

Backup and recovery obligations ensure business continuity in the event of data loss.

- **[Mandatory]** Contractors must backup.

### 3.8 Data retention

Data retention obligations set minimum and maximum retention periods for records.

- **[Recommended]** Employees should data retention.

### 3.9 Physical

Physical security obligations cover access to offices, data centers, and equipment.

- **[Recommended]** All Users should physical.

### 3.10 Network

Network obligations govern segmentation, firewalling, and perimeter defenses.

- **[Mandatory]** Developers shall network.
