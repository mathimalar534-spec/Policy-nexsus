# HR Security Policy

**Version:** v1.2  
**Owner:** Daniel Kim (Compliance)  
**Last Reviewed:** 2025-11-04  
**Status:** Retired

## 1. Purpose

This document establishes the hr security policy for the organization. It is maintained by the Compliance department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Monitoring

Monitoring obligations define what systems must be observed and alerted on.

- **[Mandatory]** Employees must monitoring.
- **[Recommended]** All Users should monitoring.

### 3.2 HR

HR obligations apply to onboarding, offboarding, and personnel security practices.

- **[Mandatory]** Contractors required HR.

### 3.3 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Recommended]** Contractors should cloud.
- **[Recommended]** Developers should cloud.

### 3.4 Access

Access control obligations define who may reach which systems and data.

- **[Mandatory]** Contractors must access. (Reference: WEP)

### 3.5 Endpoint

Endpoint obligations apply to laptops, workstations, and other managed endpoints.

- **[Recommended]** Developers recommended endpoint.

### 3.6 Change

Change management obligations govern how modifications to production systems are approved.

- **[Mandatory]** Admins shall change.

### 3.7 Network

Network obligations govern segmentation, firewalling, and perimeter defenses.

- **[Recommended]** All Users should network.
- **[Mandatory]** Developers must network.

### 3.8 Logging

Logging obligations define what activity must be recorded and for how long.

- **[Mandatory]** Contractors required logging. (Reference: NIST SP 800-53 Rev 4)
