# Data Retention Policy

**Version:** v3.5  
**Owner:** Ava Johnson (Infrastructure)  
**Last Reviewed:** 2022-09-22  
**Status:** Active

## 1. Purpose

This document establishes the data retention policy for the organization. It is maintained by the Infrastructure department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Backup

Backup and recovery obligations ensure business continuity in the event of data loss.

- **[Mandatory]** Admins required backup.

### 3.2 Patch

Patch management obligations set expectations for applying security updates.

- **[Recommended]** Admins should patch.
- **[Recommended]** All Users should patch.

### 3.3 Change

Change management obligations govern how modifications to production systems are approved.

- **[Mandatory]** All Users must change.

### 3.4 Access

Access control obligations define who may reach which systems and data.

- **[Mandatory]** All Users must access.

### 3.5 Logging

Logging obligations define what activity must be recorded and for how long.

- **[Mandatory]** Contractors required logging.

### 3.6 Provisioning

Provisioning obligations define how user accounts are created and deprovisioned.

- **[Recommended]** Contractors recommended provisioning.

### 3.7 Privacy

Privacy obligations govern the collection, use, and disclosure of personal data.

- **[Recommended]** Employees recommended privacy. (Reference: GDPR 2016)
- **[Prohibited]** Admins prohibited privacy.

### 3.8 Data retention

Data retention obligations set minimum and maximum retention periods for records.

- **[Prohibited]** Employees prohibited data retention. (Reference: Windows Server 2012)

### 3.9 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Mandatory]** Admins must cloud.

### 3.10 Mobile

Mobile device obligations apply to any device used to access company resources.

- **[Recommended]** Developers recommended mobile.
