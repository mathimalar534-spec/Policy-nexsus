# Change Management Policy

**Version:** v2.0  
**Owner:** Daniel Kim (Support)  
**Last Reviewed:** 2025-02-24  
**Status:** Active

## 1. Purpose

This document establishes the change management policy for the organization. It is maintained by the Support department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Mandatory]** Employees required cloud.

### 3.2 HR

HR obligations apply to onboarding, offboarding, and personnel security practices.

- **[Mandatory]** Admins shall HR.
- **[Mandatory]** Employees required HR.

### 3.3 Logging

Logging obligations define what activity must be recorded and for how long.

- **[Mandatory]** Employees must logging.
- **[Recommended]** Admins should logging. (Reference: NIST SP 800-53 Rev 4)

### 3.4 Physical

Physical security obligations cover access to offices, data centers, and equipment.

- **[Recommended]** Employees recommended physical.
- **[Mandatory]** All Users required physical.

### 3.5 Patch

Patch management obligations set expectations for applying security updates.

- **[Mandatory]** Developers must patch.

### 3.6 Vendor

Vendor obligations apply to the selection and oversight of external suppliers.

- **[Recommended]** Contractors recommended vendor.

### 3.7 Password

Password obligations govern how credentials are created, stored, and rotated.

- **[Mandatory]** Admins required password.

### 3.8 Provisioning

Provisioning obligations define how user accounts are created and deprovisioned.

- **[Recommended]** All Users should provisioning.

### 3.9 Mobile

Mobile device obligations apply to any device used to access company resources.

- **[Mandatory]** All Users must mobile.
