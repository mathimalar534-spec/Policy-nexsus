# Incident Response Policy

**Version:** v2.3  
**Owner:** Noah Patel (Compliance)  
**Last Reviewed:** 2025-12-12  
**Status:** Retired

## 1. Purpose

This document establishes the incident response policy for the organization. It is maintained by the Compliance department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Monitoring

Monitoring obligations define what systems must be observed and alerted on.

- **[Mandatory]** Developers shall monitoring.
- **[Mandatory]** Admins must monitoring.

### 3.2 Privacy

Privacy obligations govern the collection, use, and disclosure of personal data.

- **[Mandatory]** All Users must privacy. (Reference: WEP)

### 3.3 Logging

Logging obligations define what activity must be recorded and for how long.

- **[Recommended]** Contractors recommended logging.

### 3.4 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Mandatory]** All Users must cloud.
- **[Mandatory]** Employees must cloud.

### 3.5 Backup

Backup and recovery obligations ensure business continuity in the event of data loss.

- **[Mandatory]** Admins must backup.

### 3.6 Third-party

Third-party obligations govern engagements with vendors and external partners.

- **[Recommended]** Developers recommended third-party. (Reference: TLS 1.0)

### 3.7 Data retention

Data retention obligations set minimum and maximum retention periods for records.

- **[Mandatory]** Contractors must data retention.
- **[Recommended]** All Users should data retention.

### 3.8 Physical

Physical security obligations cover access to offices, data centers, and equipment.

- **[Recommended]** Admins should physical. (Reference: NIST SP 800-53 Rev 4)

### 3.9 HR

HR obligations apply to onboarding, offboarding, and personnel security practices.

- **[Mandatory]** Contractors must HR.
