# Mobile Device Policy

**Version:** v3.2  
**Owner:** Fatima Noor (IT Security)  
**Last Reviewed:** 2025-10-09  
**Status:** Retired

## 1. Purpose

This document establishes the mobile device policy for the organization. It is maintained by the IT Security department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Logging

Logging obligations define what activity must be recorded and for how long.

- **[Mandatory]** Admins required logging.
- **[Mandatory]** All Users must logging.

### 3.2 Provisioning

Provisioning obligations define how user accounts are created and deprovisioned.

- **[Mandatory]** Developers required provisioning.
- **[Recommended]** All Users recommended provisioning.
- **[Mandatory]** Admins shall provisioning.

### 3.3 Privacy

Privacy obligations govern the collection, use, and disclosure of personal data.

- **[Mandatory]** Admins required privacy.

### 3.4 Api

API obligations govern how internal and external interfaces must be secured.

- **[Mandatory]** Employees required api.
- **[Prohibited]** Employees prohibited api.

### 3.5 Data retention

Data retention obligations set minimum and maximum retention periods for records.

- **[Mandatory]** Developers required data retention.

### 3.6 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Recommended]** Contractors should cloud.

### 3.7 Mobile

Mobile device obligations apply to any device used to access company resources.

- **[Mandatory]** Contractors must mobile.

### 3.8 Physical

Physical security obligations cover access to offices, data centers, and equipment.

- **[Prohibited]** Developers prohibited physical.

## 4. References

This policy's technical baseline is defined with reference to Windows Server 2008 R2, as adopted at the time of the last major revision.
