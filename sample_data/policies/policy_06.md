# Network Security Policy

**Version:** v3.0  
**Owner:** Noah Patel (Operations)  
**Last Reviewed:** 2022-12-15  
**Status:** Active

## 1. Purpose

This document establishes the network security policy for the organization. It is maintained by the Operations department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Third-party

Third-party obligations govern engagements with vendors and external partners.

- **[Mandatory]** Employees shall third-party.

### 3.2 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Recommended]** All Users should cloud.

### 3.3 Mobile

Mobile device obligations apply to any device used to access company resources.

- **[Mandatory]** Developers required mobile. (Reference: Windows Server 2012)
- **[Recommended]** Contractors should mobile.
- **[Recommended]** Admins recommended mobile.

### 3.4 Api

API obligations govern how internal and external interfaces must be secured.

- **[Recommended]** Admins recommended api.

### 3.5 HR

HR obligations apply to onboarding, offboarding, and personnel security practices.

- **[Mandatory]** Admins must HR.

### 3.6 Password

Password obligations govern how credentials are created, stored, and rotated.

- **[Recommended]** Developers recommended password.

### 3.7 Backup

Backup and recovery obligations ensure business continuity in the event of data loss.

- **[Mandatory]** Developers required backup.

### 3.8 Logging

Logging obligations define what activity must be recorded and for how long.

- **[Mandatory]** Employees must logging.

### 3.9 Asset

Asset management obligations cover the tracking and disposal of company assets.

- **[Recommended]** Contractors recommended asset.

### 3.10 Vendor

Vendor obligations apply to the selection and oversight of external suppliers.

- **[Mandatory]** Employees shall vendor.
