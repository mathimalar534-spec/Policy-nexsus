# Cloud Security Policy

**Version:** v2.2  
**Owner:** Priya Nair (Sales)  
**Last Reviewed:** 2022-07-15  
**Status:** Active

## 1. Purpose

This document establishes the cloud security policy for the organization. It is maintained by the Sales department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Asset

Asset management obligations cover the tracking and disposal of company assets.

- **[Prohibited]** Developers prohibited asset.

### 3.2 Network

Network obligations govern segmentation, firewalling, and perimeter defenses.

- **[Mandatory]** Developers must network.

### 3.3 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Recommended]** Admins should cloud.

### 3.4 Password

Password obligations govern how credentials are created, stored, and rotated.

- **[Prohibited]** All Users prohibited password.

### 3.5 Privacy

Privacy obligations govern the collection, use, and disclosure of personal data.

- **[Mandatory]** Employees shall privacy.

### 3.6 Third-party

Third-party obligations govern engagements with vendors and external partners.

- **[Prohibited]** Admins prohibited third-party.

### 3.7 Provisioning

Provisioning obligations define how user accounts are created and deprovisioned.

- **[Mandatory]** Contractors required provisioning. (Reference: SOX 2002)
- **[Recommended]** Contractors recommended provisioning.

### 3.8 Change

Change management obligations govern how modifications to production systems are approved.

- **[Mandatory]** Employees required change.

### 3.9 Encryption

Encryption obligations specify how data must be protected at rest and in transit.

- **[Prohibited]** Admins prohibited encryption.

### 3.10 Mobile

Mobile device obligations apply to any device used to access company resources.

- **[Recommended]** Developers recommended mobile.

### 3.11 Monitoring

Monitoring obligations define what systems must be observed and alerted on.

- **[Mandatory]** Developers shall monitoring. (Reference: NIST SP 800-53 Rev 4)
