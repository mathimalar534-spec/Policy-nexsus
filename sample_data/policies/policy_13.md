# Physical Security Policy

**Version:** v2.0  
**Owner:** Priya Nair (Legal)  
**Last Reviewed:** 2025-08-12  
**Status:** Draft

## 1. Purpose

This document establishes the physical security policy for the organization. It is maintained by the Legal department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, contractors, developers, employees.

## 3. Obligations

### 3.1 Network

Network obligations govern segmentation, firewalling, and perimeter defenses.

- **[Mandatory]** Contractors shall network.

### 3.2 Api

API obligations govern how internal and external interfaces must be secured.

- **[Recommended]** Employees should api.

### 3.3 Endpoint

Endpoint obligations apply to laptops, workstations, and other managed endpoints.

- **[Mandatory]** Admins shall endpoint. (Reference: DES)
- **[Prohibited]** Employees prohibited endpoint.

### 3.4 Password

Password obligations govern how credentials are created, stored, and rotated.

- **[Mandatory]** Developers shall password.

### 3.5 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Mandatory]** Employees must cloud.
- **[Recommended]** Admins should cloud.

### 3.6 Patch

Patch management obligations set expectations for applying security updates.

- **[Mandatory]** Admins shall patch.

### 3.7 HR

HR obligations apply to onboarding, offboarding, and personnel security practices.

- **[Recommended]** Developers should HR.

### 3.8 Asset

Asset management obligations cover the tracking and disposal of company assets.

- **[Mandatory]** Developers required asset.

### 3.9 Change

Change management obligations govern how modifications to production systems are approved.

- **[Recommended]** Employees should change.
- **[Recommended]** Employees should change.
