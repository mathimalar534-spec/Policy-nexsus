# API Security Policy

**Version:** v1.2  
**Owner:** Alex Morgan (Infrastructure)  
**Last Reviewed:** 2025-03-16  
**Status:** Retired

## 1. Purpose

This document establishes the api security policy for the organization. It is maintained by the Infrastructure department and applies to all personnel and systems in scope below.

## 2. Scope

This policy applies to the following groups: admins, all users, contractors, developers, employees.

## 3. Obligations

### 3.1 Api

API obligations govern how internal and external interfaces must be secured.

- **[Recommended]** Developers recommended api.

### 3.2 Cloud

Cloud obligations apply to infrastructure and services hosted with cloud providers.

- **[Prohibited]** Contractors prohibited cloud.
- **[Recommended]** Employees should cloud.

### 3.3 Vendor

Vendor obligations apply to the selection and oversight of external suppliers.

- **[Prohibited]** Employees prohibited vendor.

### 3.4 Privacy

Privacy obligations govern the collection, use, and disclosure of personal data.

- **[Mandatory]** Admins shall privacy.
- **[Mandatory]** Contractors must privacy.

### 3.5 Third-party

Third-party obligations govern engagements with vendors and external partners.

- **[Mandatory]** Employees must third-party.

### 3.6 Asset

Asset management obligations cover the tracking and disposal of company assets.

- **[Recommended]** Contractors recommended asset.
- **[Recommended]** Admins recommended asset.

### 3.7 Provisioning

Provisioning obligations define how user accounts are created and deprovisioned.

- **[Recommended]** All Users should provisioning.

### 3.8 Mobile

Mobile device obligations apply to any device used to access company resources.

- **[Prohibited]** All Users prohibited mobile.
